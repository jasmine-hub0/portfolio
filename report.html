HTML Website Project Report

Jasmine Khadka

Project Summary

This project involved designing and building a personal portfolio website using semantic HTML5 only, without any CSS styling. The website consists of four interconnected pages: Home (index.html), About, Projects, and Contact. The goal was to apply semantic tags properly to enhance accessibility, structure, and clarity.

Development Process

  1. Planning: Created a sitemap and outlined the content hierarchy to ensure logical page flow and user-friendly navigation.
  2. Home Page: Developed the landing page introducing myself with a welcoming message and navigation links to other pages.
  3. About Page: Added background information including education and interests, organized with semantic sections.
  4. Projects Page: Listed and described sample projects using articles with titles, descriptions, and links to live demos.
  5. Contact Page: Provided contact details and links to social profiles formatted using lists for clarity and accessibility.
  6. Validation and Testing: Checked all HTML files for proper semantic markup, working links, and consistent structure across pages.

Reflection

This project reinforced the value of semantic HTML5 in building accessible and well-structured web pages. It also improved my understanding of multi-page site navigation and linking without relying on stylesheets, laying a solid foundation for future web development projects.
